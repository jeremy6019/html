
** XHTML
1.HTML
=>웹 브라우저의 표현하는 문서의 구조를 만들기 위해서 만든 언어 
=> <태그 속성 = '값'>내용</태그> 또는 <태그 속성="값"/>로 표현하도록 규칙
을 정함 
=>해석을 브라우저에서 담당 
=>엄격한 문법을 지원하지 않고 느슨한 형태의 문법을 지원을 해서 브라우저가 
자신의 편의에 맞추어서 해석 
=>중복해서 태그를 만들때 닫는 순서를 지키지 않아도 브라우저가 알아서 해석 

<div><a href=""></a>,/div>
<div><a href=""></div></a> : 이 경우도 가능  
<div><div></div> : 이 경우도 가능 
=> 동일한 코드에 대해서 해석하는 브라우저마다 다르게 표현 되기도 합니다. 

2.XML 
=>확장 마크업 언어 
=>웹에서 데이터를 표현하기 위한 언어 
=>문법이 엄격해서 태그의 닫는 순서라던가 열면 반드시 닫아야 한다던가 
속성은 반드시 문자열로 입력해야 합니다. 
=>동일한 문장으로 만들어 진것은 동일하게 해석해야 하기 때문입니다. 

3.XHTML
=>HTML에 XML의 엄격한 문법을 적용한 버전으로 HTML4.01이라고도 합니다.
=>요즈음 HTML이라고 하면 XHTML을 의미합니다. 

4.XHTML의 구성요소 
1) 태그: <와>사이에 묶어서 표현 
기본형식은 <tag>내용</tag>또는 <tag/>로 표현 

2)속성(attribute): 태그 안에 작성해서 태그의 개별적인 옵션을 설정 
<tag 속성명 ='값' ...>으로 작성 
값을 작성할 때 ' '또는 " "안에 작성 
대부분의 경우는 따옴표를 하지않아도 적용은 됩니다. 

3)모든 태그에 사용할 수 있는 공통 속성 
=>id:자바스크립트에서 태그를 구분해서 사용하기 위한 속성 
이 속성의 값은 현재 페이지 내에서 중복되면 안됩니다. 

=>clss:css에서 공통된 디자인을 적용하기 위한 속성 
이 속성의 값은 중복 가능 

=>name:서버에게 값을 전달하기 위한 속성 
특별한 경우(체크박스, 라디오버튼 등)에 중복이 가능 
대신에 value속성을 이용해서 구분 

5.서식 
1) 태그를 만들때 종료 요소 생략 불가능 
<tag>-</tag> 또는 </tag> 
=>HTML5에서는 몇몇 태그에 한해서 생략이 가능 

2) 태그 이름과 속성이름은 소문자로 작성 

3) 속성의 값 생략 불가능 


4) 잘못된 중첩은 불가능 

5) 속성의 값을 입력할 때 따옴표 생략은 불가능 

6) 태그 관련된 문자는 변환해서 사용 
< : &lt ;
>: &gt;
&: &amp;
공백: &nbsp;

** XHTML요소 
1. meta: 문서의 정보를 나타내는 태그 
1) 문자열 기본 설정: <meta charset="인코딩 설정">
인코딩: 문자를 컴퓨터에 저장할 때 숫자로 변환하는 방법 
디코딩: 숫자로 저장되어 있는 문자를 화면에 출력하기 위해서 변환하는 방법 

영문과 숫자는 어떤 인코딩 방식에서건 숫자 값이 같습니다.  
한글은 3가지 정도 방법이 있는데 윈도우즈 응용프로그램에서 사용하는 방식은 
ms949(cp949)라고 하는 방식이 있고 예전의 웹 프로그램에서 한들과 영문과 숫자만
표현하던 euc-kr이라는 방식이 있고 전 세계 모든 글자를 전부 표현하는 utf-8이라는
인코딩 방식이 있습니다. 
인코딩 방식이 다르면 한글이 제대로 표현이 되지 않으므로 인코딩방식을 변환해보아야 
합니다.
외국에서 만든 응용 프로그램을 사용할 때 한글지원 여부를 확인해야 합니다. 
MySQL은 기본 인코딩이 iso-9958-1이어서 기본적으로 한글이 지원이 안됩니다. 
테이블을 만들 때 접속할 때 인코딩 방식을 설정해 주어야 합니다.

1)shape: rect, circle, poly(다각형)

2)coords 
rect는 왼쪽상단 x좌표, 왼쪽상단y좌표, 오른쪽 하단 x좌표, 오른쪽하단 y좌표

circle은 중점의 x좌표, 중점의 y좌표, 반지름 

poly는 x좌표와 y좌표를 순서대로 기입 

6.table태그 
1)table
=>테이블을 생성하는 태그 

2)tr 
=>하나의 행을 생성하는 태그 

3)td
=>하나의 셀을 생성하는 태그 

4)th
=>제목셀을 생성하는 태그 
=>글자에 b태그가 적용되고 글자가 셀에 가운데 맞춤 


5)border속성 
=>table에만 적용이 가능한데 경계선 두께 설정으로 기본값은 0 
=>bordercolor속성을 이용해서 경계선의 색상 설정 가능 
=>border-style속성을 이용해서 선 모양 설정 가능 

=>style="border:두께 단위 선모양 색상"
선모양으로는 nonr, dotted, dashed, solid, double등 

6)여백 속성 - table에서만 적용 
cellspacing:셀간의 여백 
cellpadding:셀 내에서 콘텐츠와의 여백 
(padding은 border와 콘텐츠와의 여백이고 margin은 다른 콘텐츠와의 여백)

7)맞춤 속성(align)- table, tr, td, th모두 사용가능 
=>table에 사용하면 테이블 전체가 맞춤 
=>th나 td에 사용하면 콘텐츠가 맞춤 

8) 크기 조정 - table, tr, td, th모두 사용가능 
=> width 는 가로크기 
=>height는 세로크기 
=>가로크기는 첫번째 줄에서만 설정하면 모든 행에 동일한 크기가 적용 
=>셀별로 따로 지정하고자 하면 셀마다 전부 설정해야합니다.

9)배경 - bgcolor로 설정하는데 모든 태그에 적용가능 
색상명은 알려진색상은 색상이름을 그대로 기재 
#rrggbb
#rgb
=>16진수로 설정하는데 2자리 씩 6자리로 설정할 수 있고1자리씩 3개를 설정
할 수 있는데 예전에는 1자리씩 3개를 입력했습니다 
1자리씩 3개 입력하면 각 자리를 복사해서 사용 
#369 -> #336699가 됩니다. 

10)셀 합치기 
colspan: 옆으로 셀을 합치는 것 

rowspan: 세로로 셀을 합치는 것 

=>합쳐줄 개수를 대입 
=>합친 후 셀의 배치를 잘못하면 셀이 안보여지거나 테이블이 깨지는 현상이 발생 
=>rowspan이 된 경우에는 다음 행을 만들 때 한 개를 제외하고 셀을 생성해야 합니다. 

11)caption 
=>table태그안에 별도로 작성해서 표의 제목을 나타내는 태그 
=>음성 브라우저의 경우 table과 연관있다고 읽어줍니다. 

12)table의 summary속성 
=>테이블에 대한 요약 내용을 작성하는 속성 
=>음성브라우저에서는 읽어주기 때문에 내용을 듣고 건너뛰도록 할 수 있습니다. 

13)thead, tbody, tfoot
=>table태그에서는 기본적으로 tr순서에 맞추어서 배치 
=>tr태그를 thead로 감싸면 무조건 가장 상단에 위치
=>tr태그를 tfoot로 감싸면 무조건 가장 하단에 위치
=>thead로 감쌀 때는 td대신에 tr을 사용해야 합니다. 

=>thead태그로 감싸면 내용을 읽을때마다 같이 읽어 줍니다.
